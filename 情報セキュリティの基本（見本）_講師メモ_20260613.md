# 情報セキュリティの基本（見本）　講師メモ
合計 約45分 ／ 6枚

### [1] はじめての情報セキュリティ　（表紙・2分）
- 要点：新卒エンジニア向けワークショップ / 〜守る人になるための第一歩〜
- 🎙 トーク：明るく自己紹介し、「今日は怖がらせる時間ではなく、守り方を身につける時間です」と伝える。

### [2] なぜセキュリティが必要？　（導入・6分）
- 要点：情報は会社の大切な財産 / 一度の漏えいが信頼を失わせる / 守るのは特別な人ではなく全員
- 🎙 トーク：身近なニュースを1つ挙げ、自分ごととして考えてもらう。

### [3] 3つの基本：機密性・完全性・可用性　（本編・10分）
- 要点：機密性：見せていい人にだけ見せる / 完全性：勝手に書き換えられない / 可用性：使いたいときに使える
- 🎙 トーク：うさうさラーメン店にたとえる：レシピは秘密(機密性)、味を変えない(完全性)、開店時間に開ける(可用性)。

### [4] やってみよう：強いパスワード　（演習・12分）
- 要点：弱い例と強い例を見比べる / パスフレーズの作り方を体験
- 🎙 トーク：恥ずかしい実例より、作る楽しさにフォーカスする。
- 🛠 ワーク：3〜4人グループで「覚えやすく破られにくいパスフレーズ」を1つ考え、なぜ強いかを1分で発表する。

### [5] このメール、開いていい？　（クイズ・8分）
- 要点：3問の○×クイズ / フィッシングの見抜き方
- 🎙 トーク：挙手やチャットで全員参加。間違えても責めない雰囲気づくりを。
- 🛠 ワーク：差出人・URL・添付ファイルの3点を確認するクイズを出題し、判断の理由を共有する。

### [6] 今日からできる3つの習慣　（まとめ・7分）
- 要点：怪しいと思ったら一人で抱えず相談 / パスワードは使い回さない / 困ったらすぐ報連相
- 🎙 トーク：完璧でなくていい、相談できる人が一番強い、と締める。Q&Aへ。
